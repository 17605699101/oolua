12 /*
13 Extract from the Lua programming Reference 5.1
14 2.8 - Metatables
15 
16 Every value in Lua may have a metatable. This metatable is an ordinary Lua
17 table that defines the behavior of the original value under certain special
18 operations. You can change several aspects of the behavior of operations
19 over a value by setting specific fields in its metatable. For instance,
20 when a non-numeric value is the operand of an addition, Lua checks for a
21 function in the field "__add" in its metatable. If it finds one, Lua calls
22 this function to perform the addition.
23 
24 We call the keys in a metatable events and the values metamethods. In the
25 previous example, the event is "add" and the metamethod is the function that
26 performs the addition.
27 
28 You can query the metatable of any value through the getmetatable function.
29 
30 You can replace the metatable of tables through the setmetatable function.
31 You cannot change the metatable of other types from Lua (except using the
32 debug library); you must use the C API for that.
33 
34 Tables and full userdata have individual metatables (although multiple
35 tables and userdata can share their metatables); values of all other types
36 share one single metatable per type. So, there is one single metatable for
37 all numbers, one for all strings, etc.
38 
39 A metatable may control how an object behaves in arithmetic operations,
40 order comparisons, concatenation, length operation, and indexing. A metatable
41 can also define a function to be called when a userdata is garbage collected.
42 For each of these operations Lua associates a specific key called an event.
43 When Lua performs one of these operations over a value, it checks whether
44 this value has a metatable with the corresponding event. If so, the value
45 associated with that key (the metamethod) controls how Lua will perform the
46 operation.
47 
48 Metatables control the operations listed next. Each operation is identified
49 by its corresponding name. The key for each operation is a string with its
50 name prefixed by two underscores, '__'; for instance, the key for operation
51 "add" is the string "__add". The semantics of these operations is better
52 explained by a Lua function describing how the interpreter executes the operation.
53 */
